# 技術スタック (Tech Stack)

## 概要
本ウェブサイトは、静的なコンテンツを配信するためのシンプルな構成を採用しています。主にフロントエンド技術と、開発・デバッグを補助するツールで構成されています。

## 詳細

### 1. マークアップ言語
*   **HTML5**: 最新のウェブ標準に準拠したマークアップ言語が使用されています。

### 2. スタイルシート
*   **Tailwind CSS (CDN)**: ユーティリティファーストのCSSフレームワークがCDN経由で利用されています。これにより、迅速なUI開発とレスポンシブデザインの実現が可能です。
*   **arakaki.css**: プロジェクト固有のカスタムスタイルが定義されています。Tailwind CSSのユーティリティクラスでは表現しきれない、より詳細なスタイリングやオーバーライドに使用されています。

### 3. スクリプト言語
*   **バニラJavaScript (arakaki.js)**: 特定のフレームワークやライブラリに依存しない純粋なJavaScriptが使用されています。スライドショー機能や、SPA（シングルページアプリケーション）風のセクション切り替えロジック（現在は一部修正済み）など、動的な要素の実装に用いられています。

### 4. フォント
*   **Google Fonts**:
    *   `Inter`: 主に英数字の表示に使用されていると推測されます。
    *   `Noto Serif JP`: 日本語の表示に適したセリフ体フォントが使用されており、ウェブサイトの視認性とデザイン性を向上させています。

### 5. 画像リソース (開発・プレースホルダー)
*   **placehold.co**: 開発中に一時的な画像プレースホルダーとして利用されています。
*   **api.qrserver.com**: QRコードの生成サービスとして利用されています。
*   **ローカル画像ファイル**: `こだわり用フォルダー` (現在は`kodawari`) や `ホームスライド用フォルダー` など、プロジェクト内に直接配置された画像ファイルが使用されています。

### 6. 開発環境
*   **Pythonの `http.server`**: ローカルでのウェブサイト開発およびテストのために、シンプルなHTTPサーバーとして利用されています。

## 技術スタックの最適性評価

### 良い点
*   **シンプルさと軽量性**: 静的サイトであるため、複雑なバックエンドやデータベースは不要であり、非常にシンプルで軽量な構成です。
*   **迅速な開発**: Tailwind CSSの採用により、CSSを記述する手間が省け、UIのプロトタイピングや開発が迅速に行えます。
*   **CDNの活用**: Tailwind CSSやGoogle FontsをCDN経由で利用することで、初期設定の手間を減らし、グローバルな配信速度を向上させています。
*   **バニラJavaScript**: 小規模な動的機能であれば、余分なオーバーヘッドなしに実装できます。

### 改善点・考慮事項
*   **Tailwind CSSの最適化**: CDN経由での利用は手軽ですが、本番環境ではPurgeCSSなどを用いて未使用のCSSを削除し、ファイルサイズを最適化することが望ましいです。ビルドプロセスを導入することで実現できます。
*   **JavaScriptの管理**: `arakaki.js`は現在、グローバルスコープで動作しています。機能が複雑化した場合、モジュール化やバンドラー（Webpack, Rollup, Viteなど）の導入を検討することで、コードの保守性やパフォーマンスを向上できます。
*   **SPAロジックと複数ページ構成の混在**: `arakaki.js`のSPA風ナビゲーションロジックが、複数HTMLファイル構成と混在しており、過去に問題を引き起こしました。サイトの規模や要件に応じて、完全にSPAに移行するか、純粋な複数ページ構成に統一するかを明確にすることが重要です。
*   **画像パスの管理**: ローカルの絶対パスの使用は避けるべきです。相対パスを使用し、本番環境へのデプロイを考慮したパス設計が必要です。
*   **開発サーバー**: Pythonの`http.server`は手軽ですが、より高機能な開発サーバー（ホットリロード、プロキシ機能など）を必要とする場合は、Node.jsベースのツール（Webpack Dev Server, Viteなど）の導入も検討できます。
*   **SEO**: 静的サイトジェネレーター（Jekyll, Hugo, Next.jsのSSGなど）を導入することで、コンテンツ管理を容易にしつつ、SEOに強い静的サイトを構築することも可能です。

## 結論
現在の技術スタックは、**小規模な静的ウェブサイトの迅速な開発とデプロイには非常に適した、シンプルで効率的な構成**であると評価できます。

ただし、サイトの規模拡大や機能追加、パフォーマンスのさらなる最適化を目指す場合は、上記「改善点・考慮事項」で挙げたようなビルドツールやフレームワークの導入を検討することで、より堅牢で保守性の高いシステムへと発展させることが可能です。
